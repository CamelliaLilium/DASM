usage: domain_gap_calculator.py [-h] [--pkl_path PKL_PATH]
                                [--test_data_root TEST_DATA_ROOT]
                                [--embedding_rates EMBEDDING_RATES [EMBEDDING_RATES ...]]
                                [--output_dir OUTPUT_DIR] [--epochs EPOCHS]
                                [--batch_size BATCH_SIZE] [--lr LR]
                                [--d_model D_MODEL] [--num_heads NUM_HEADS]
                                [--num_layers NUM_LAYERS] [--dropout DROPOUT]
                                [--device DEVICE] [--gpu GPU] [--seed SEED]
                                [--balance_classes]

Domain Gap Calculator using PAD

options:
  -h, --help            show this help message and exit
  --pkl_path PKL_PATH   Path to combined dataset PKL file
  --test_data_root TEST_DATA_ROOT
                        Test data root directory
  --embedding_rates EMBEDDING_RATES [EMBEDDING_RATES ...]
                        Embedding rates to test (e.g., --embedding_rates 0.1
                        0.5 1.0)
  --output_dir OUTPUT_DIR
                        Output directory for results
  --epochs EPOCHS       Training epochs
  --batch_size BATCH_SIZE
                        Batch size
  --lr LR               Learning rate
  --d_model D_MODEL     Transformer model dimension
  --num_heads NUM_HEADS
                        Number of attention heads
  --num_layers NUM_LAYERS
                        Number of transformer layers
  --dropout DROPOUT     Dropout rate
  --device DEVICE       Device (cuda/cpu)
  --gpu GPU             GPU ID to use (default: 0)
  --seed SEED           Random seed
  --balance_classes     Balance class samples (each class will have same
                        number of samples as the minority class). If not set,
                        use imbalanced data directly.
